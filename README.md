# PROJECT TITLE :
 PEER-TO-PEER RESOURCE HUB (sycet-013m)

 # WEBAPP NAME :
 CircleShare

# DESCRIPTIION : 
CircleShare is a web platform designed to eliminate the struggle of finding realiable study notes,previous year question paper (PYQ'S), and reference materials . unlike a simple file sharing folder, this hub uses a social ranking system to ensure the most accurate and helpful content is easiest to find . it fosters a culture of collaborative learning where peers help each other by passing and sharing of curated resources.  

# SOLUTION :
 To build an effective solution ,we focused on:
    
    1) Easy to upload and download of a study materials.
    2) Doubt session. 
    3) Organised year-wise and subject-wise content
    4) Fast search functionality.
    5) Secure students login systems

# CORE FEATURES :

 1) STRUCTURED UPLOAD :
 
                      
                      Users are categorized by 
                      department wise (e.g,
                      computer science,mechanical,
                      electrical,5g,ece,civil 
                      etc),resource type,and 
                      specific subject codes to 
                      keep the data based 
                      organized.

 2) QUALITY RANKING SYSTEM :
                    
                      Highly-rated notes appear
                      first in search results, 
                      while low-quality or 
                      incorrect files are flagged

 3) PREVIEW MODE :
 
                    
                    Integration of a PDF views so 
                    users can read the first few 
                    pages before deciding to 
                    download the file

 4) USER DASHBOARD : 
 
                   
                   A personal space for 
                   contributors to see how many 
                   people have been helped by 
                   their notes (gamification).

 # TECHNICAL IMPLEMENTATION :

   FRONTEND: HTML, CSS(Tailwind), JavaScript

   BACKEND :  Firebase/ Node.js

   DATABASE : Firestore

   STORAGE :  cloudinary
